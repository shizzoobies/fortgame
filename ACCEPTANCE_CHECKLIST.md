# ACCEPTANCE_CHECKLIST.md

A build is acceptable when all of the following are true:

- The game runs by opening `index.html`
- No build tooling is required
- Fortress auto-attacks enemies
- At least 3 enemy types are implemented
- Waves can be started and completed
- Gold is awarded correctly
- At least 4 upgrades work
- Upgrade costs scale
- Fortress can be damaged and destroyed
- Game over screen appears
- Restart works without reloading page
- UI clearly shows wave, HP, and gold
- Visuals are coherent and intentional
- No external libraries are required
- Code is readable enough for further iteration
